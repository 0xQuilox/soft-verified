This POC tests Key Management, Recovery Logic, and Transaction Signing/Broadcasting mechanisms.

Status: Ready to test. Make sure the Verified Wallet extension is installed.

1. Key Management & Recovery Logic Tests

Test 1.1: Request Private Key (getPk)

Tests the requestPk method which should trigger a popup to request private key access.

Test 1.2: Sign Recovery Transaction (signRecovery)

Tests the signRecovery method used in wallet recovery flows.

Test 1.3: Complete Recovery (completeRecovery)

Tests the completeRecovery method to complete wallet recovery.

Test 1.4: Check localStorage for Key Exposure

Checks if sensitive key material is stored in localStorage (accessible from web pages).

2. Transaction Signing & Broadcasting Tests

Test 2.1: Send Transaction (eth_sendTransaction)

Tests the eth_sendTransaction method to sign and broadcast transactions.

Test 2.2: Send Transaction (sendTransaction)

Tests the alternative sendTransaction method.

Test 2.3: Message Flow Analysis

Monitors message passing between web page, content script, and background script.

Test 2.4: Origin Validation Test

Tests if the extension properly validates message origins (potential vulnerability).

3. Comprehensive Test Suite

Run all tests sequentially to identify vulnerabilities.
